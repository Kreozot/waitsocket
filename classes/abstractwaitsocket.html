sendRequest

  • sendRequest<RequestPayloadType, ResponsePayloadType>(type: string, payload?: RequestPayloadType, waitForType?: string, requestJSONSchema?: SchemaObject, responseJSONSchema?: SchemaObject): Promise<{ message: string; payload: ResponsePayloadType }>
    • Defined in AbstractWaitSocket.ts:249

    Send message to server and asynchronously wait for the response. By default it adds random "requestId" parameter to "meta" section of the message. In order to identify the response as relative to the specific request, server must return back this "meta.requestId" in the response message. If server not returns "meta.requestId", you can specify "waitForType" parameter: In that case the response will be identified by "type" in the response message.

    example
    // Recommended way
const {payload, message} =
  await sendRequest('MY_REQUEST', { optionalPayload: 'example' });
// Outgoing message will be {
//   type: 'MY_REQUEST',
//   payload: { optionalPayload: 'example' },
//   meta: { requestId: 'nb1SQCTRmbDSv1u4idPr1' }
// }
    example
    // Not recommended way
const {payload, message} =
  await sendRequest('MY_REQUEST', { optionalPayload: 'example' }, 'MY_RESPONSE');
// Outgoing message will be {
//   type: 'MY_REQUEST',
//   payload: {optionalPayload: 'example'},
// }

    Type parameters

    • RequestPayloadType = any

    • ResponsePayloadType = any

    Parameters

    • type: string

      Message type identifier

    • Optional payload: RequestPayloadType

      Message payload

    • Optional waitForType: string

      Message type in the response waiting for (not recommended)

    • Optional requestJSONSchema: SchemaObject

      JSONSchema object to validate request message before sending

    • Optional responseJSONSchema: SchemaObject

      JSONSchema object to validate response message after receiving

    Returns Promise<{ message: string; payload: ResponsePayloadType }>

    } Object with payload and raw message string
